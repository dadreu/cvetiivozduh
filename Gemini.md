# Память Gemini: Архитектура и Дизайн Цветочного Магазина

Данный файл содержит ключевые архитектурные решения и правила, которых необходимо придерживаться при разработке.

## 1. Технологический стек
*   **Бэкенд:** C# (.NET 8/9), Minimal APIs. Никаких избыточных слоев (Clean Architecture, полноформатный Identity). Подход Vertical Slice.
*   **База данных:** MySQL. ORM: Entity Framework Core + провайдер `Pomelo.EntityFrameworkCore.MySql`. Code-First миграции.
*   **Фронтенд:** Vue 3 (Composition API), Vite. Статический пререндеринг для SEO публичного каталога.
*   **Инфраструктура:** Nginx (Reverse Proxy, раздача статики, SSL), RuVDS Linux, Systemd (демон для Kestrel).

## 2. Правила Бэкенда (C#)
*   **Minimal APIs:** Регистрация маршрутов и middleware в строгом порядке (Exceptions -> Routing -> Forwarded Headers -> Authentication -> Authorization). Маршруты чувствительны к регистру.
*   **База данных:**
    *   Пул контекстов: `AddDbContextPool`.
    *   Автоопределение версии: `ServerVersion.AutoDetect(connectionString)`.
    *   Запросы на чтение (публичный каталог) строго с `.AsNoTracking()`.
    *   Все операции I/O асинхронны.
    *   Использование DTO, а не сырых сущностей (Entities).
*   **Аутентификация:** Cookie-Based (не JWT в LocalStorage!).
    *   CookieOptions: `HttpOnly = true`, `Secure = true`, `SameSite = Strict/Lax`, `IsEssential = true`.
    *   Data Protection: ключи должны сохраняться на диск `PersistKeysToFileSystem`, чтобы не слетала сессия при перезапуске сервера.
*   **Загрузка файлов (изображения):**
    *   Ограничение размера: настройка `MaxRequestBodySize` и `MultipartBodyLengthLimit`.
    *   Изображения сохраняются на диск (например, `/var/www/flowershop/uploads`), а не в БД. Nginx будет раздавать их напрямую.
    *   Имена файлов генерируются сервером (`Guid.NewGuid()`). Оригинальные имена игнорируются.
    *   Обязательная настройка Anti-forgery токенов для загрузки.

## 3. Правила Фронтенда (Vue 3)
*   **Архитектура:** Composition API, Pinia (стейт), Vue Router.
*   **SEO:** Использование `vite-plugin-prerender` для лендинга. Подавление Hydration Mismatch на динамических элементах (`data-allow-mismatch`).
*   **Интеграция с API:**
    *   Axios с обязательным параметром `withCredentials: true` для отправки куки.
    *   Интерцепторы ответов (401 -> очистка стейта Pinia и редирект на логин).
    *   Передача Anti-forgery токена (`X-CSRF-TOKEN`) при POST-запросах.
*   **Дизайн (Скандинавский минимализм):**
    *   Отказ от визуального шума, "воздух" (негативное пространство).
    *   **Базовый цвет (60%):** Теплый белый, кремовый, светло-серый (#F9F9F6, #F5F5F5).
    *   **Вторичный цвет (30%):** Пыльный синий, шалфейный зеленый, теплый беж.
    *   **Акцентный цвет (10%):** Терракотовый, глубокий синий, графитовый черный (#2C2C2C). Кнопки CTA.
    *   Высокий контраст (доступность), плавная анимация.

## 4. Антипаттерны (Строго избегать!)
*   Overengineering бэкенда (микросервисы, CQRS, лишние репозитории поверх EF).
*   Хранение JWT в LocalStorage браузера.
*   Возврат EF-моделей напрямую через API (только DTO).
*   Публикация Kestrel напрямую в интернет (без Nginx).
